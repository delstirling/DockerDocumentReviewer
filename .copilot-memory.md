## 2026-05-21
- Implemented organization-scoped tier access with two tiers: `admin`, `user`.
- Added `users.organization_tier` field in schema and migration `drizzle/0002_organization_tier.sql`.
- Updated auth/session payloads to include `organizationTier`.
- Extended `/api/organization`:
  - `GET` returns current user's organization and member list for that organization only.
  - `POST` creates organization and auto-assigns creator as `admin`.
  - `POST` with `action: addMember` allows admin to add existing users by email.
  - `PATCH` allows admin to change member tier.
  - `DELETE` allows admin to remove members.
  - Enforced guardrail: cannot remove/demote the last admin.
- Enhanced organization page UI with admin member management controls (add/remove users + tier dropdown), while non-admins have read-only organization/member visibility.
